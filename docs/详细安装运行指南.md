## 系统要求

### 必需软件
- **Node.js**: 版本 14.0 或更高（推荐 16.x 或 18.x）
- **npm**: 通常随 Node.js 一起安装（版本 6.0 或更高）
- **操作系统**: Windows 7/10/11, macOS, Linux

### 硬件要求
- **内存**: 最低 2GB RAM（推荐 4GB 或更多）
- **硬盘**: 至少 500MB 可用空间
- **网络**: 需要互联网连接（仅安装时）

### 浏览器支持
- Chrome 90+
- Firefox 88+
- Edge 90+
- Safari 14+

## 安装前准备

### 1️⃣ 检查 Node.js 是否已安装

打开命令提示符（Windows）或终端（Mac/Linux），输入：

```bash
node --version
npm --version
```

**预期输出示例：**
```
v16.14.0
8.3.1
```

如果显示版本号，说明已安装 ✅

### 2️⃣ 如果未安装 Node.js

#### Windows 用户：
1. 访问 https://nodejs.org/
2. 下载 LTS（长期支持）版本
3. 双击安装包，按提示完成安装
4. 重启命令提示符，再次检查版本

#### Mac 用户：
```bash
# 使用 Homebrew 安装
brew install node
```

#### Linux 用户：
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nodejs npm

# CentOS/RHEL
sudo yum install nodejs npm
```

### 3️⃣ 下载项目文件

确保您已将项目文件夹 `pet-health-system` 放置在合适的位置，例如：
- Windows: `C:\Users\您的用户名\Desktop\pet-health-system`
- Mac/Linux: `~/Desktop/pet-health-system`

## 详细安装步骤

### 方法一：使用自动化脚本（推荐，Windows）

#### 步骤 1: 运行安装脚本

1. 打开项目文件夹 `pet-health-system`
2. 双击运行 `安装依赖.bat`
3. 等待安装完成（可能需要 5-10 分钟）

**安装过程中会看到：**
```
正在安装后端依赖...
正在安装前端依赖...
安装完成！
```

#### 步骤 2: 启动所有服务

1. 双击运行 `启动所有服务.bat`
2. 等待两个命令行窗口打开（一个后端，一个前端）

---

### 方法二：手动安装（适用于所有系统）

#### 步骤 1: 安装后端依赖

**Windows:**
```cmd
cd C:\Users\您的用户名\Desktop\pet-health-system\backend
npm install
```

**Mac/Linux:**
```bash
cd ~/Desktop/pet-health-system/backend
npm install
```

**预期输出：**
```
added 150 packages in 30s
```

#### 步骤 2: 配置后端环境变量

1. 进入 `backend` 文件夹
2. 复制 `.env.example` 并重命名为 `.env`

**Windows:**
```cmd
cd backend
copy .env.example .env
```

**Mac/Linux:**
```bash
cd backend
cp .env.example .env
```

3. （可选）编辑 `.env` 文件自定义配置：
```env
PORT=3001
JWT_SECRET=your-secret-key-change-this-in-production
DATABASE_PATH=./pet_health.db
UPLOAD_DIR=./uploads
```

#### 步骤 3: 初始化数据库

**数据库会在首次启动时自动创建**，无需手动操作。

但如果需要重置数据库：
1. 删除 `backend/pet_health.db`
2. 重新启动后端服务

#### 步骤 4: 安装前端依赖

**Windows:**
```cmd
cd C:\Users\您的用户名\Desktop\pet-health-system\frontend
npm install
```

**Mac/Linux:**
```bash
cd ~/Desktop/pet-health-system/frontend
npm install
```

**预期输出：**
```
added 200 packages in 40s
```

## 常见问题解决

### ❌ 问题 1: 端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3001
```

**解决方案 A - 更改端口：**

1. 后端端口冲突：
   - 编辑 `backend/.env`
   - 修改 `PORT=3001` 为 `PORT=3002`
   - 同时修改前端 API 配置 `frontend/src/services/api.js` 中的 `baseURL`

2. 前端端口冲突：
   - 编辑 `frontend/vite.config.js`
   - 修改 `server.port` 为其他端口（如 3005）

**解决方案 B - 关闭占用端口的程序：**

**Windows:**
```cmd
# 查找占用端口的进程
netstat -ano | findstr :3001

# 结束进程（替换 PID）
taskkill /PID 进程ID /F
```

**Mac/Linux:**
```bash
# 查找并结束进程
lsof -ti:3001 | xargs kill -9
```

---

### ❌ 问题 2: npm install 失败

**错误信息：**
```
npm ERR! network timeout
```

**解决方案：**

1. 使用淘宝镜像加速：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

2. 清除缓存后重试：
```bash
npm cache clean --force
npm install
```

3. 使用 `--legacy-peer-deps` 参数：
```bash
npm install --legacy-peer-deps
```

---

### ❌ 问题 3: 数据库错误

**错误信息：**
```
Error: SQLITE_ERROR: no such table: users
```

**解决方案：**

1. 删除现有数据库：
```bash
# 进入 backend 目录
cd backend
# Windows
del pet_health.db
# Mac/Linux
rm pet_health.db
```

2. 重新启动后端服务，数据库会自动重建

---

### ❌ 问题 4: 跨域错误 (CORS)

**错误信息（浏览器控制台）：**
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解决方案：**

1. 确认后端已启动
2. 检查 `backend/server.js` 中的 CORS 配置：
```javascript
app.use(cors({
  origin: 'http://localhost:3000',
  credentials: true
}));
```

3. 如果前端端口不是 3000，修改 `origin` 值

---

### ❌ 问题 5: 图片上传失败

**解决方案：**

1. 检查 `backend/uploads` 文件夹是否存在
2. 如果不存在，手动创建：
```bash
cd backend
mkdir uploads
```

3. 检查文件大小是否超过 5MB
4. 检查文件格式是否为 JPG/PNG/GIF

---

### ❌ 问题 6: 无法登录/Token 过期

**解决方案：**

1. 清除浏览器本地存储：
   - 打开浏览器开发者工具 (F12)
   - 进入 Application/存储 标签
   - 清除 localStorage
   - 刷新页面

2. 重新注册/登录

---

### ❌ 问题 7: 前端页面空白

**解决方案：**

1. 打开浏览器控制台 (F12) 查看错误
2. 确认后端服务已启动
3. 清除浏览器缓存并刷新 (Ctrl+Shift+R)
4. 检查 `frontend/src/services/api.js` 中的 API 地址是否正确

## 使用教程

### 第一次使用流程

#### 1. 注册账户
- 访问 http://localhost:3000
- 点击"注册"链接
- 填写用户名、邮箱、密码（至少6位）
- 点击"注册"按钮

#### 2. 登录系统
- 使用刚才注册的用户名/邮箱和密码登录
- 登录成功后会跳转到宠物列表页面

#### 3. 添加第一只宠物
1. 点击页面右上角的"添加宠物"按钮
2. 上传宠物照片：
   - 点击头像上传区域
   - 选择图片文件（JPG/PNG/GIF，最大5MB）
   - 预览确认
3. 填写宠物信息：
   - 名字：例如 "小白"
   - 种类：狗/猫/兔子等
   - 品种：例如 "金毛"
   - 性别：公/母
   - 出生日期：选择日期
   - 体重：单位kg
   - 主人姓名
   - 联系电话
4. 点击"保存"按钮

#### 4. 查看宠物详情
- 在宠物列表中点击宠物卡片
- 查看完整的宠物信息
- 查看健康记录和疫苗记录

#### 5. 添加健康记录
1. 进入宠物详情页
2. 点击"添加健康记录"
3. 填写：
   - 记录类型：常规检查/疾病/受伤等
   - 记录日期
   - 体重、体温
   - 描述和备注
4. 保存

#### 6. 添加疫苗记录
1. 在宠物详情页点击"疫苗记录"标签
2. 点击"添加疫苗记录"
3. 填写：
   - 疫苗名称：例如 "狂犬疫苗"
   - 接种日期
   - 下次接种日期
   - 兽医姓名
   - 备注
4. 保存

#### 7. 查看数据统计
1. 点击导航栏的"数据统计"
2. 选择要查看的宠物
3. 切换时间维度（日/周/月）
4. 查看：
   - 健康趋势图（体重、体温变化）
   - 活动量分布
   - 健康概览统计

#### 8. 编辑宠物信息
1. 在宠物列表点击"编辑"按钮
2. 修改需要更新的信息
3. 点击"保存"

#### 9. 删除宠物
1. 在宠物列表点击"删除"按钮
2. 确认删除操作
3. ⚠️ 注意：删除操作不可恢复

## 数据管理

### 数据存储位置

#### 数据库文件：
```
backend/pet_health.db
```
- 包含所有用户、宠物、健康记录

#### 上传的图片：
```
backend/uploads/
```
- 存储所有宠物头像

### 备份数据

**重要：定期备份以防数据丢失！**

#### Windows:
```cmd
# 备份数据库
copy backend\pet_health.db backup\pet_health_备份_20240411.db

# 备份图片
xcopy backend\uploads backup\uploads /E /I
```

#### Mac/Linux:
```bash
# 备份数据库
cp backend/pet_health.db backup/pet_health_备份_20240411.db

# 备份图片
cp -r backend/uploads backup/uploads
```

### 恢复数据

1. 停止后端服务
2. 替换数据库文件：
```bash
# 删除当前数据库
rm backend/pet_health.db
# 恢复备份
cp backup/pet_health_备份_20240411.db backend/pet_health.db
```
3. 恢复图片文件夹
4. 重启后端服务

### 重置系统（清空所有数据）

⚠️ **警告：此操作会删除所有数据，无法恢复！**

1. 停止前后端服务
2. 删除数据库：
```bash
cd backend
rm pet_health.db
```
3. 清空上传文件夹：
```bash
rm -rf uploads/*
mkdir uploads
```
4. 重新启动服务

## 开发调试

### 查看日志

#### 后端日志：
- 查看启动后端的命令行窗口

### 测试 API 接口

使用 Postman 或 curl 测试：

```bash
# 注册用户
curl -X POST http://localhost:3001/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@example.com","password":"123456"}'

# 登录获取 Token
curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'

# 获取宠物列表（需要替换 YOUR_TOKEN）
curl http://localhost:3001/api/pets \
  -H "Authorization: Bearer YOUR_TOKEN"
```
