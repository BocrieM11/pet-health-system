## 🧪 测试清单

### ✅ 测试1：提醒系统（3分钟）

**步骤：**

1. **访问提醒页面**
   - 登录系统
   - 点击导航栏 **🔔 提醒**

2. **创建测试提醒**
   - 点击 **"+ 新建提醒"**
   - 填写信息：
     ```
     宠物：选择任意宠物（可选）
     提醒类型：自定义
     标题：测试提醒
     描述：这是一个测试提醒
     提醒时间：选择5分钟后的时间
     ```
   - 点击 **"创建提醒"**

3. **验证功能**
   - ✅ 提醒出现在列表中
   - ✅ 状态显示为"待发送"
   - ✅ 可以筛选状态
   - ✅ 可以标记完成/忽略
   - ✅ 可以删除

4. **测试自动提醒**
   - 进入任意宠物详情页
   - 添加疫苗记录，填写"下次接种日期"（选择未来某天）
   - 等待第二天上午9点，系统会自动创建提醒

**预期结果：**
- ✅ 提醒创建成功
- ✅ 5分钟后状态自动变为"已发送"
- ✅ 后端日志显示：`🔔 检查待处理的提醒...`

### ✅ 测试3：性能优化（2分钟）

**步骤：**

1. **查看数据库索引**
   - 检查后端启动日志
   - 确认看到：`✅ 数据库表和索引初始化完成`

2. **测试响应压缩**
   - 打开浏览器开发者工具（F12）
   - 切换到 Network 标签
   - 刷新宠物列表页面
   - 查看 API 请求的 Size 列
   - 应该看到 gzip 压缩标识

3. **测试分页查询**
   - 如果健康记录超过20条：
   - 打开 Network 标签
   - 查看 `/api/health/pet/:id` 请求
   - 响应应该包含 pagination 字段

**预期结果：**
- ✅ 页面加载速度明显提升
- ✅ API响应时间减少
- ✅ 响应体积变小（有gzip标识）

**对比方法：**
- 查看 Network 标签中的 Time 列
- 查看 Size 列（transferred / size）

## 🐛 问题排查

### 提醒系统问题

**问题：提醒没有自动变为"已发送"**

检查清单：
- [ ] 后端是否正常运行？
- [ ] 提醒时间是否已过？
- [ ] 后端日志是否显示定时任务？
- [ ] 刷新页面查看最新状态

**解决方法：**
```bash
# 重启后端
cd backend
npm run dev

# 查看日志，确认看到：
✅ 提醒定时任务已启动
```

---

### 导出功能问题

**问题：点击导出没反应**

检查清单：
- [ ] 是否已登录？
- [ ] 浏览器控制台是否有错误？
- [ ] 后端服务是否正常？
- [ ] 是否有宠物数据？

**解决方法：**
1. 打开浏览器控制台（F12）
2. 查看 Console 标签的错误信息
3. 查看 Network 标签的请求状态

**常见错误：**
```
401 Unauthorized - 未登录或Token过期
404 Not Found - 宠物不存在或无权访问
500 Server Error - 后端错误（查看后端日志）
```

---

### 性能问题

**问题：查询仍然很慢**

检查清单：
- [ ] 数据库索引是否创建成功？
- [ ] 数据量是否过大（>10000条）？
- [ ] 是否使用了分页？

**解决方法：**

1. **验证索引：**
```bash
# 停止后端
# 删除数据库（注意备份！）
cd backend
del pet_health.db  # Windows
rm pet_health.db   # Mac/Linux

# 重新启动，会自动创建索引
npm run dev
```

2. **清理数据：**
```sql
-- 删除2年前的健康记录
DELETE FROM health_records 
WHERE record_date < date('now', '-2 years');
```
