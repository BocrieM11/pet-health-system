# 第一部分：项目概述

## 1.1 项目背景与需求分析

### 背景介绍
随着人们生活水平的提高，宠物饲养已经成为现代家庭的重要组成部分。据统计，中国的宠物数量已超过1亿只。然而，在宠物健康管理方面，我们发现了几个突出的问题：

**问题一：信息记录分散**
传统的纸质记录方式容易丢失，宠物主人很难系统地追踪宠物的健康历史。比如疫苗接种记录、体检报告等重要信息常常找不到。

**问题二：数据缺乏分析**
即使有记录，这些数据也是孤立的，无法进行趋势分析。比如宠物体重是否正常增长？体温变化是否有规律？这些问题很难通过零散的记录得到答案。

**问题三：提醒功能缺失**
宠物需要定期接种疫苗、体检、驱虫等，但很多主人容易忘记这些重要时间节点。

**问题四：专业性不足**
普通宠物主人缺乏专业知识，很难判断宠物的健康状况是否正常。

### 核心需求
基于以上问题，我们提炼出了以下核心需求：

1. **信息集中管理** - 将所有宠物信息集中在一个平台，随时随地可查询
2. **数据可视化** - 将健康数据以图表形式展示，直观了解趋势
3. **智能分析** - 利用AI技术提供专业的健康分析建议
4. **自动提醒** - 定时提醒重要的健康管理事项
5. **数据导出** - 支持生成报告，便于与兽医沟通
6. **多宠物管理** - 支持同时管理多只宠物

```
│         SQLite Database             │
│         文件：pet_health.db         │
```

### 通信流程

**典型的请求-响应流程：**

1. **用户操作** → 用户在浏览器点击"查看宠物详情"
2. **前端请求** → React组件调用API服务：`petAPI.getPetById(id)`
3. **HTTP请求** → Axios发送GET请求：`GET /api/pets/:id`
4. **JWT验证** → 后端中间件验证Token有效性
5. **业务处理** → Express路由处理请求，查询数据库
6. **数据库查询** → SQLite执行SQL：`SELECT * FROM pets WHERE id = ?`
7. **返回响应** → JSON格式数据返回给前端
8. **页面渲染** → React组件更新状态，重新渲染UI

### 技术选型理由

**为什么选择React？**
- 组件化开发，代码复用率高
- 虚拟DOM机制，性能优秀
- 生态系统丰富（Router、图表库等）
- 学习曲线相对平缓
- 社区活跃，问题容易解决

**为什么选择Express？**
- 极简灵活，不限制开发方式
- 中间件机制强大，易于扩展
- Node.js天然支持异步，处理并发能力强
- 与前端JavaScript技术栈统一
- 轻量级，适合中小型项目

**为什么选择SQLite？**
- 零配置，无需安装数据库服务器
- 文件型数据库，便于备份和迁移
- 支持完整的SQL标准
- 性能优秀（对于单机应用）
- 适合开发和演示环境

## 3.2 前端核心功能实现

### 3.2.1 用户认证流程

**JWT Token自动管理**

使用Axios拦截器实现Token的自动注入和失效处理：

```javascript
// src/services/api.js

// 请求拦截器：自动添加Token
api.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token');
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  error => Promise.reject(error)
);

// 响应拦截器：处理401未授权
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      // Token过期或无效，清除并跳转登录
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

**实现效果：**
- 用户登录后，Token自动存储到localStorage
- 每次API请求自动携带Token，无需手动处理
- Token失效时自动跳转登录页，用户体验流畅

**登录流程：**
1. 用户输入用户名和密码
2. 前端调用`authAPI.login(username, password)`
3. 后端验证成功，返回JWT Token
4. 前端存储Token到localStorage
5. 跳转到首页（宠物列表）

### 3.2.2 API服务封装

为了代码的可维护性，我们将所有API调用封装到`api.js`文件中：

```javascript
// src/services/api.js

import axios from 'axios';

const api = axios.create({
  baseURL: '/api',  // Vite代理到后端
  timeout: 10000
});

// API方法按模块分组
export const authAPI = {
  login: (username, password) => 
    api.post('/auth/login', { username, password }),
  register: (username, email, password) => 
    api.post('/auth/register', { username, email, password })
};

export const petAPI = {
  getAllPets: () => api.get('/pets'),
  getPetById: (id) => api.get(`/pets/${id}`),
  createPet: (petData) => api.post('/pets', petData),
  updatePet: (id, petData) => api.put(`/pets/${id}`, petData),
  deletePet: (id) => api.delete(`/pets/${id}`)
};

export const healthAPI = {
  getHealthRecords: (petId) => api.get(`/health/records/${petId}`),
  createHealthRecord: (recordData) => api.post('/health/records', recordData),
```

### 4.2.1 JWT认证机制

**Token生成：**

```javascript
// routes/auth.js - 登录接口

const jwt = require('jsonwebtoken');
const bcrypt = require('bcryptjs');

router.post('/login', async (req, res) => {
  const { username, password } = req.body;
  
  try {
    // 1. 查询用户
    const user = await db.get(
      'SELECT * FROM users WHERE username = ?',
      [username]
    );
    
    if (!user) {
      return res.status(401).json({ error: '用户名或密码错误' });
    }
    
    // 2. 验证密码
    const isValid = await bcrypt.compare(password, user.password);
    if (!isValid) {
      return res.status(401).json({ error: '用户名或密码错误' });
    }
    
    // 3. 生成JWT Token（7天有效期）
    const token = jwt.sign(
      { 
        userId: user.id, 
        username: user.username 
      },
      process.env.JWT_SECRET || 'your-secret-key',
      { expiresIn: '7d' }
    );
    
    // 4. 返回Token
    res.json({ 
      token, 
      user: { 
        id: user.id, 
        username: user.username, 
        email: user.email 
      } 
    });
    
  } catch (error) {
    console.error('登录失败:', error);
    res.status(500).json({ error: '服务器错误' });
  }
});
```

**Token验证中间件：**

```javascript
// middleware/auth.js

const jwt = require('jsonwebtoken');

const authenticateToken = (req, res, next) => {
  // 1. 从请求头获取Token
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1]; // Bearer TOKEN
  
  if (!token) {
    return res.status(401).json({ error: '未提供认证令牌' });
  }
  
  // 2. 验证Token
  jwt.verify(token, process.env.JWT_SECRET || 'your-secret-key', (err, user) => {
    if (err) {
      return res.status(403).json({ error: '令牌无效或已过期' });
    }
    
    // 3. 将用户信息附加到请求对象
    req.user = user;
    next();
  });
};

module.exports = { authenticateToken };
```

**使用示例：**

```javascript

  );

```

- ✅ 密码bcrypt加密存储（10轮盐值）

- ✅ Token有效期7天，过期自动失效

```javascript
    cb(null, 'uploads/');  // 上传目录

    // 生成唯一文件名：时间戳-随机数.扩展名
    const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);

const fileFilter = (req, file, cb) => {
  const allowedTypes = /jpeg|jpg|png|gif/;
  const extname = allowedTypes.test(path.extname(file.originalname).toLowerCase());
  const mimetype = allowedTypes.test(file.mimetype);
  
  if (extname && mimetype) {
    return cb(null, true);
  } else {
    cb(new Error('只支持JPG、PNG、GIF格式的图片'));
  }
};

// 3. 创建upload实例
const upload = multer({
  storage: storage,
  limits: { 
    fileSize: 5 * 1024 * 1024  // 5MB限制
  },
  fileFilter: fileFilter
});

// 4. 上传路由
router.post('/avatar', upload.single('avatar'), (req, res) => {
  if (!req.file) {
    return res.status(400).json({ error: '未上传文件' });

  const fileUrl = `/uploads/${req.file.filename}`;
  res.json({ 
    message: '上传成功', 
    url: fileUrl 
  });
});

// 5. 静态文件服务（在server.js中）
app.use('/uploads', express.static('uploads'));
```

```javascript
// 规则引擎降级方案
function ruleBasedAnalysis(pet, records) {
  let score = 100;
  const warnings = [];
  const suggestions = [];
  
  // 体重分析
  const avgWeight = records.reduce((sum, r) => sum + (r.weight || 0), 0) / records.length;
  if (avgWeight > pet.weight * 1.2) {
    score -= 15;
    warnings.push('体重增长过快');
    suggestions.push('建议控制饮食，增加运动');
  }
  
  // 体温分析
  const latestTemp = records[0].temperature;
  if (latestTemp > 39.5) {
    score -= 20;
    warnings.push('体温偏高');
    suggestions.push('建议尽快就医检查');
  }
  
  // 更多规则...
  
  return { score, warnings, suggestions, method: 'rule-based' };
}
```

```javascript
async function processReminders() {

    // 查询待处理的提醒（scheduled_time <= 当前时间）
    const reminders = await db.all(`
      SELECT r.*, u.email, p.name as pet_name
      FROM reminders r
      JOIN users u ON r.user_id = u.id
```

## 4.3 数据库Promise封装

SQLite3原生API是基于回调的，我们封装成Promise：

```javascript
// database.js

const sqlite3 = require('sqlite3').verbose();
const path = require('path');

const dbPath = path.resolve(__dirname, 'pet_health.db');
const db = new sqlite3.Database(dbPath);

// 封装Promise方法
const dbAsync = {
  // 查询单行
  get: (sql, params = []) => {
    return new Promise((resolve, reject) => {
      db.get(sql, params, (err, row) => {
        if (err) reject(err);
        else resolve(row);
      });
    });
  },
  
  // 查询多行
  all: (sql, params = []) => {
    return new Promise((resolve, reject) => {
      db.all(sql, params, (err, rows) => {
        if (err) reject(err);
        else resolve(rows);
      });
    });
  },
  
  // 执行SQL（INSERT/UPDATE/DELETE）
  run: (sql, params = []) => {
    return new Promise((resolve, reject) => {
      db.run(sql, params, function(err) {
        if (err) reject(err);
        else resolve({ lastID: this.lastID, changes: this.changes });
      });
    });
  }
};

module.exports = dbAsync;
```

## 5.2 数据库优化策略

### 索引优化

**1. 单列索引**
- `users.username` - 登录查询
- `users.email` - 注册验证
- `pets.user_id` - 用户宠物列表

**2. 复合索引**
- `health_records(pet_id, record_date DESC)` - 按日期查询健康记录
- `reminders(scheduled_time, status)` - 定时任务查询

**索引选择原则：**
- WHERE条件中的列
- JOIN连接的外键
- ORDER BY排序的列
- 查询频率高的列

### 查询优化

**使用参数化查询（防止SQL注入）：**

```javascript
// ✅ 正确
await db.get('SELECT * FROM users WHERE username = ?', [username]);

// ❌ 错误（SQL注入风险）
await db.get(`SELECT * FROM users WHERE username = '${username}'`);
```

**批量操作优化：**

```javascript
// ❌ 低效：多次单条插入
for (const record of records) {
  await db.run('INSERT INTO health_records (...) VALUES (...)', [record.data]);
}

// ✅ 高效：批量插入
const values = records.map(r => `(${r.data})`).join(',');
await db.run(`INSERT INTO health_records (...) VALUES ${values}`);
```

### 性能监控

```javascript
// 查询执行时间监控
const startTime = Date.now();
```

## Q&A 常见问题准备

### 技术问题

**Q1: 为什么选择SQLite而不是MySQL/PostgreSQL？**
A: SQLite适合中小规模应用，零配置、文件型数据库便于部署和演示。如果需要扩展，可以轻松迁移到其他数据库。

**Q2: JWT有什么优势？**
A: 无状态认证，服务器无需存储Session，支持分布式部署，Token包含用户信息减少数据库查询。

**Q3: 如何保证API安全？**
A: 使用JWT验证身份，bcrypt加密密码，参数化查询防止SQL注入，数据隔离保证用户只能访问自己的数据。

**Q4: AI分析的准确性如何保证？**
A: 使用Anthropic最先进的Claude模型，并结合规则引擎双重验证，同时保留原始数据供用户和兽医判断。

**Q5: 性能如何优化？**
A: 数据库索引优化、Promise并行请求、图表按需加载、响应压缩、静态资源缓存。

### 功能问题

**Q6: 支持多少只宠物？**
A: 无限制，用户可以管理任意数量的宠物。

**Q7: 数据会丢失吗？**
A: SQLite数据库文件持久化存储，建议定期备份。未来会支持云端同步。

**Q8: 能否导入历史数据？**
A: 当前版本需要手动录入，未来版本将支持Excel批量导入。

**Q9: 提醒功能需要联网吗？**
A: 定时任务在后端服务器运行，需要保持后端服务运行。未来移动端会支持本地推送。

**Q10: 支持团队协作吗？**
A: 当前版本是单用户系统，未来会支持家庭成员共同管理。
